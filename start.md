# アイデア
## 日常の体験を溜める
* 普段利用するサービスの感想を溜めていく
 * これは体験して体に溜めていく
 * 例）Dungeon raid のパズル感が手軽で楽しい！
 * 例）ポケモンのモンスター育成と収集が楽しい！

## アイデアをまとめる
* 思いついたアイデアを溜めていく
 * evernoteや紙のノートに書き溜める
 * 例）Dungeon raidのパズルにくっつくRPGをポケモンにしたら面白そう。※絵も描く
* 溜まったアイデアをA4１枚の紙に書きまとめる
 * 書き溜めたアイデアを取捨選択しながら形成する
 * 例）メインとなる画面を描く。周りにそれに付随する機能や特徴や面白い点、ユーザーの声を描く。

# プロダクト
## アイデアの核となるユーザーフローを発見
* 利用者のプロダクトを使う核となるユーザーフロー図をつくる
 * 例）start -> パズルゲーム -> モンスター獲得 -> モンスター育成 -> パズルゲームに戻る
* 核となるユーザーフロー図にあるそれぞれの項目を機能にしていく
 * 例）モンスター獲得：パズルゲームでのドロップ、イベントパズルゲームでのドロップ、無料ガチャでの獲得、課金ガチャでの獲得など

## プロダクトの核となるものをプロトタイプ化
* 核となる機能をしぼり、プロトタイプ化する
 * 例）楽しいパズルゲームとなるか簡単なプロトタイプをつくる
 * 仮定や理想から外れなければ、このまま進める。外れれば、ユーザーフロー図の作り直しもしくは、破棄。

## プロダクトを作っていく
* 機能をさらに細かくしていく
 * 機能毎に細かくユーザーストーリーを書いていく
 * 機能が大きすぎる場合は、親子関係にある機能を洗い出し、ユーザーストーリーを書いていく
 * ユーザーストーリーは、「○○が▲▲できる。★★のため。」形式で書く
 * 例）モンスター合成：プレイヤーがベースとなるモンスターを選択できる。プレイヤー自身の成長させたいと思っているモンスターを自由に選べるようにするため。
* ユーザーストーリーをまとめて、機能を表現していく
 * デザイン・開発の工程に移行する
 * 例）モンスター合成のモンスター選択画面：必要なストーリーから優先度の高いものが実現しやすい様にデザイン・開発
